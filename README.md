# AstrBot 智能提醒插件 (smart_reminder)

三大功能可独立开关的 AstrBot 插件：日常提醒（含人名模糊识别@）、游戏组局召集/上号提醒、生日祝贺。

## 功能一览

### 功能开关

插件提供三个独立开关，在 WebUI 配置面板中可分别控制：

| 开关 | 配置项 | 默认 | 说明 |
|------|--------|------|------|
| 日常提醒 | `enable_reminder` | ✅ 开启 | 普通提醒、人名模糊识别、指令、后台轮询 |
| 游戏提醒 | `enable_gaming` | ✅ 开启 | 游戏召集、上号提醒的 LLM 工具和上下文注入 |
| 生日祝福 | `enable_birthday` | ✅ 开启 | 生日当天零点自动祝贺 |

关闭某个开关后，对应的 LLM 工具、上下文注入、后台轮询均不执行，互不影响。

### 一、日常提醒
- **自然语言设置提醒**：直接告诉机器人"提醒我30分钟后抢票"、"明天下午3点提醒我开会"
- **引用消息设置提醒**：引用群里有人说"星期六两点在新街口见"的消息，然后说"提前两小时提醒我们"
- **人名模糊识别 @**：说"提醒啊哈上线"，机器人从别名对照表识别"啊哈"→QQ号，提醒触发时自动 @ 对方
- **指令方式**：`/remind set`、`/remind list`、`/remind cancel`

### 二、游戏提醒
- **组局召集**：@机器人说"海不海"/"瓦不瓦"，机器人自动 @ 对应游戏分组的群友，LLM 生成召集文案
- **模糊匹配**：别名关键词（如"海克斯""大乱斗""无畏契约"）也能触发
- **额外指定人**：说"小明、小红海不海"，除了 @ 分组成员，还会额外 @ 指定的人
- **上号提醒**：引用时间消息 @机器人说"提醒大家上号"，自动提前 5 分钟提醒
- **分组可配置**：WebUI 中配置分组名称、QQ号列表、别名关键词

### 三、生日祝福
- **零点自动发送**：生日当天零点自动在指定群发送祝贺
- **@ 寿星**：文案中 `[at:{qq}]` 占位符自动转为真实 @
- **WebUI 配置名单**：QQ号 + 生日(MM-DD) + 目标群号
- **预设文案可自定义**：支持 `{qq}` 和 `[at:{qq}]` 占位符

## 安装

1. 将 `astrbot_plugin_smart_reminder` 目录上传至 AstrBot 的 `data/plugins/` 目录
2. 在 AstrBot WebUI 插件管理页面重载插件
3. 在插件配置中按需开启/关闭功能，填入游戏分组、生日名单、联系人别名等

也可在 AstrBot WebUI 插件管理页面直接通过 GitHub 仓库链接安装。

## 使用方式

### 日常提醒

| 示例 | 说明 |
|------|------|
| "提醒我30分钟后抢票" | 相对时间提醒，@ 设置者 |
| "提醒我明天下午3点开会" | 绝对时间提醒，@ 设置者 |
| 引用消息 + "提前两小时提醒我们" | 引用消息提醒，@ 设置者 |
| "提醒啊哈上线" | 人名模糊识别，提醒触发时 @ "啊哈"对应的QQ号 |

### 游戏提醒 - 组局召集

| 示例 | 说明 |
|------|------|
| @机器人 "海不海" | 召集海克斯大乱斗分组群友 |
| @机器人 "今晚瓦一把" | 召集无畏契约分组群友 |
| @机器人 "小明、小红海不海" | 召集分组群友 + 额外 @ 小明和小红 |
| @机器人 "帮我问下今天有没有人海" | 模糊匹配触发召集 |

### 游戏提醒 - 上号提醒

1. 群友回复时间（如"七点半"）
2. 你引用"七点半"这条消息，@机器人说"到时候提醒大家上号"
3. 机器人提前 5 分钟在群里发送预设提醒文案

### 生日祝福

自动触发，无需操作。配置好名单后，生日当天零点自动发送。

## 配置

在 AstrBot WebUI 插件配置中可调整。以下配置项均使用 `template_list` 类型，在 WebUI 中以模板化列表形式可视化编辑。

### 功能开关

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enable_reminder` | true | 启用日常提醒 |
| `enable_gaming` | true | 启用游戏提醒 |
| `enable_birthday` | true | 启用生日祝福 |

### 全局配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `poll_interval` | 30 | 轮询间隔（秒） |
| `timezone` | Asia/Shanghai | 时区 |
| `enable_llm_tool` | true | 是否启用 LLM 工具 |
| `inject_time_context` | true | 是否注入时间上下文 |

### 日常提醒配置（仅在 enable_reminder 开启时生效）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `contact_aliases` | [] | 联系人称呼别名表（template_list，每项含 qq + aliases） |

### 游戏提醒配置（仅在 enable_gaming 开启时生效）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `gaming_groups` | [] | 游戏分组配置（template_list，每项含 group_name + qq_list + aliases） |
| `reminder_advance_minutes` | 5 | 上号提醒提前量（分钟） |

### 生日祝福配置（仅在 enable_birthday 开启时生效）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `birthday_list` | [] | 生日名单（template_list，每项含 qq + birthday + group_id） |
| `birthday_message` | 见下方 | 生日祝贺文案 |
| `birthday_send_hour` | 0 | 发送时间（小时，0=零点） |

### contact_aliases 配置

在 WebUI 中以模板列表形式添加联系人：

- **QQ号**：联系人的QQ号
- **称呼别名列表**：该联系人的所有称呼，如 ["小A", "AA", "啊哈"]

配置数据示例：
```json
[
  {"__template_key": "default", "qq": "12345", "aliases": ["小A", "AA", "啊哈"]},
  {"__template_key": "default", "qq": "67890", "aliases": ["大B", "BB", "波波"]}
]
```

当用户说"提醒啊哈上线"时，LLM 会从别名对照表识别"啊哈"→QQ号，在 `set_reminder` 工具的 `target_qq` 参数中填入对应QQ号，提醒触发时自动 @ 该人。

### gaming_groups 配置

在 WebUI 中以模板列表形式添加游戏分组：

- **分组名称**：如"海""瓦"，用户说"海不海"时匹配此名称
- **QQ号列表**：该分组包含的所有QQ号，召集时自动 @ 这些人
- **别名/关键词列表**：用于模糊匹配，如 ["海克斯", "大乱斗", "开海"]

配置数据示例：
```json
[
  {"__template_key": "default", "group_name": "海", "qq_list": ["123456", "789012", "345678"], "aliases": ["海克斯", "大乱斗", "海一把", "开海"]},
  {"__template_key": "default", "group_name": "瓦", "qq_list": ["123456", "456789"], "aliases": ["无畏契约", "瓦罗兰特", "瓦一把", "打瓦"]}
]
```

### birthday_list 配置

在 WebUI 中以模板列表形式添加生日祝贺：

- **QQ号**：过生日的人的QQ号
- **生日日期**：MM-DD 格式，如"01-15"
- **发送祝贺的目标群号**：祝贺消息发送到哪个群

配置数据示例：
```json
[
  {"__template_key": "default", "qq": "123456", "birthday": "01-15", "group_id": "987654321"},
  {"__template_key": "default", "qq": "789012", "birthday": "07-20", "group_id": "987654321"}
]
```

### birthday_message 默认文案

```
🎂 今天是 [at:{qq}] 的生日！祝你生日快乐，新的一年冒险等阶大涨！
```

## 注意事项

1. **LLM 模型要求**：人名识别、游戏召集和提醒均依赖 LLM 的 Tool Call 能力，建议使用 GPT-4o、DeepSeek-V3 等支持 Function Calling 的模型。

2. **人名模糊识别原理**：别名对照表在每次 LLM 请求时注入到上下文中，LLM 根据对照表将称呼映射到 QQ 号并传给工具参数。别名越多，识别越可靠。别名在游戏召集（`extra_names` 参数）和游戏上号提醒（`target_qq` 参数）中也生效。

3. **必须 @ 机器人**：游戏召集功能只有在 @ 机器人的情况下才会触发，避免"上海天气"等误触发。日常提醒不限，私聊也可用。

4. **提醒 @ 优先级**：如果有 `target_qq`（指定了被提醒人），提醒触发时 @ 该人；如果没有，群聊中 @ 设置者；私聊中不 @。

5. **生日祝贺零点触发**：生日发送时间默认为零点（00:00），轮询会在设定时间前后 1 分钟内触发。如果 `poll_interval` 较大，可能错过窗口。

6. **功能开关联动**：关闭某功能后，对应 LLM 工具返回"功能未启用"提示，上下文注入也只注入已开启功能的说明，后台轮询跳过对应检查。

7. **数据存储**：
   - `smart_reminder/reminders.json`：提醒数据
   - `smart_reminder/birthday_sent.json`：已发送生日记录（防重复）

## 支持的时间格式

- `2026-07-19 12:00`
- `2026/07/19 12:00`
- `2026年7月19日 12:00`
- `07-19 12:00`（自动补充今年）
- `7月19日 12:00`（自动补充今年）
